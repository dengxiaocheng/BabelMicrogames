# 计划目录

这个目录用于存放按用户要求新建的当前执行计划。

适用范围：

- 当前阶段的实现计划
- 运维改造计划
- 节点迁移计划
- Windows / Termux 本地入口整理计划
- 需要先落文件、再持续迭代的短中期计划

约定：

- 计划文件默认命名为 `PLAN_<topic>.md`
- 一个明确主题对应一份主计划文件，避免在聊天里长期悬空
- 这里放的是当前活跃计划，不是 canonical 架构文档
- 设计稳定并被吸收后，应同步回 `docs/`，而不是让 `plan/` 变成永久事实源

和其他目录的边界：

- `plan/`
  当前会话或当前阶段直接执行用的计划文件

- `docs/planning/`
  较早期的规划输入、参考草案和历史规划资料

- `docs/operations/`
  已经收束成运维规范的入口、脚本归档规则和节点操作文档
