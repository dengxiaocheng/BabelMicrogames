# 架构模块

这个模块回答的是系统“是什么、要变成什么、为什么这么收敛”。

适用问题：

- 当前实现基线是什么
- 长期目标架构是什么
- 应继承哪些产品能力
- 当前阶段按什么路线推进

推荐读取顺序：

1. [../ARCHITECTURE.md](../ARCHITECTURE.md)
   当前实现基线与标准执行流
2. [../SYSTEM_TARGET_ARCHITECTURE.md](../SYSTEM_TARGET_ARCHITECTURE.md)
   长期目标系统形态
3. [../FEATURE_INHERITANCE.md](../FEATURE_INHERITANCE.md)
   能力范围与继承边界
4. [../ROADMAP.md](../ROADMAP.md)
   阶段顺序与近期队列

按需读取规则：

- 只回答“现在怎么实现”时，通常只读 `ARCHITECTURE.md`
- 只回答“最终要长成什么”时，通常只读 `SYSTEM_TARGET_ARCHITECTURE.md`
- 只回答“做哪些、不做哪些”时，通常只读 `FEATURE_INHERITANCE.md`
- 只回答“下一步做什么”时，通常只读 `ROADMAP.md`
